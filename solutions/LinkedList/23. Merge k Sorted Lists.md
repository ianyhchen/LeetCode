# LeetCode 23. Merge k Sorted Lists 核心筆記

## 1. 題目解析
- **核心目標**：將 $k$ 個已按「升序」排列的鏈結串列（Linked List）合併成一個完整的升序鏈結串列。
- **輸入特性**：`lists` 是一個包含多個頭節點的陣列，每個串列長度不一。
- **關鍵點**：如何高效地在多個（最多 $k$ 個）候選節點中不斷找出最小值。

---

## 2. 核心演算法與資料結構
- **Min-Heap (優先權佇列)**：利用小頂堆動態維護各串列當前頭節點，保證每次選出最小值僅需 $O(\log k)$。
- **Divide and Conquer (分治法)**：借鑑歸併排序（Merge Sort）思想，將 $k$ 個串列對半拆解合併，將合併輪數控制在 $\log k$。
- **Dummy Node**：輔助建立結果串列，省去處理「第一個節點是誰」的複雜邏輯。

---

## 3. 解題思路與方法比較

| 特性 | 方法 A：Min-Heap (優先權佇列) | 方法 B：Divide and Conquer (分治法) |
| :--- | :--- | :--- |
| **邏輯核心** | 維護大小為 $k$ 的堆疊，每次取最小並推入其 next。 | 將陣列對半切，遞迴合併左半與右半結果。 |
| **時間複雜度** | $O(N \log k)$ | $O(N \log k)$ |
| **空間複雜度** | $O(k)$ (堆疊需儲存 $k$ 個節點) | $O(\log k)$ (遞迴呼叫的堆疊深度) |
| **適用場景** | 適合**流式數據 (Streaming)**，無法預知總量時。 | 適合**平行運算**，可將子任務分配給不同核心。 |
| **面試優勢** | 邏輯直觀，實作較快且不易出錯。 | 展現對遞迴與複雜度優化的深刻理解。 |

---

## 5. 常見錯誤與陷阱
- **Python Heap 比較異常**：
    - 在 Python 中，`heapq` 存入 tuple `(val, node)` 時，若 `val` 相同會比較 `node` 物件，引發 `TypeError`。
    - **對策**：存入 `(val, index, node)`，利用唯一的 `index` 作為排序的第二順位（Tie-breaker）。
- **空輸入與邊界處理**：
    - 必須考慮 `lists = []` 或 `lists = [None]` 的情況，這在分治法的 `mid` 計算或 Heap 的初始化中容易噴錯。
- **效率陷阱 (Linear Merge)**：
    - 嚴禁使用「第一個併第二個，結果再併第三個」的線性做法，其時間複雜度為 $O(N \cdot k)$，在測試資料量大時會導致超時 (TLE)。
- **斷鏈問題**：
    - 合併兩個串列後，若其中一個已用完，別忘了將另一個串列的剩餘部分直接接在結尾（`curr.next = l1 or l2`）。
