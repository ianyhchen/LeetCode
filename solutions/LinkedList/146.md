# LeetCode 146. LRU Cache 筆記

## 1. 題目解析
* 目標：設計一個資料結構，支援 `get` 與 `put` 操作，且時間複雜度皆為 $O(1)$。
* 核心邏輯：當快取滿了，必須移除「最久未使用」的資料。

## 2. 核心演算法與資料結構
* **Hash Map (Dictionary)**: 用於快速查找 `key` 對應的節點位置。
* **Doubly Linked List (雙向鏈結串列)**: 用於維護使用順序。
    * 靠近 `head`：最近使用的資料 (MRU)。
    * 靠近 `tail`：最久未使用的資料 (LRU)。

## 3. 解題思路與方法比較
* **Dummy Nodes 設計**: 
    * 使用 `head` 與 `tail` 兩個虛擬節點作為邊界。
    * **優點**: 所有的實體節點操作（新增/刪除）都不需要檢查 `None` 指標，邏輯高度統一且不易出錯。
* **操作流程**:
    * `get`: 存在則移到 `head` 並回傳；不存在回傳 -1。
    * `put`: 存在則更新並移到 `head`；不存在則判斷容量，必要時刪除 `tail.prev` 節點，再將新節點插入 `head`。

## 4. 常見錯誤與陷阱
* **指標更新順序**: 在 `set_head` 時，需先連接新節點與後方節點，再更新 `head.next`，否則會斷鍊。
* **Map 與 List 不同步**: 刪除節點時，務必同時從 `cache_map` 中刪除對應的 `key`。
* **忘記更新 Value**: `put` 已存在的 `key` 時，除了移動位置，也要更新節點內的 `value`。
