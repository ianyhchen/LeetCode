# LeetCode 322. Coin Change 筆記

## 1. 題目解析
- **目標**：給定不同面額 `coins`，求湊齊總額 `amount` 的**最少**硬幣個數。
- **類型**：完全背包問題 (Unbounded Knapsack) / 動態規劃。
- **回傳值**：若無法湊成則回傳 -1。

## 2. 核心演算法與資料結構
- **演算法**：動態規劃 (Bottom-up DP)
- **資料結構**：一維陣列 `dp`，長度為 `amount + 1`。

## 3. 解題思路與方法比較
- **狀態定義**：`dp[i]` 為湊齊金額 `i` 所需的最少硬幣數。
- **轉移方程**：$dp[i] = \min(dp[i], dp[i - \text{coin}] + 1)$
- **方法對比**：
    - **暴力 DFS**：$O(c^n)$，會逾時。
    - **Top-down (Memoization)**：邏輯直覺，但要注意遞迴深度限制（Stack Overflow）。
    - **Bottom-up (Iteration)**：$O(n \times k)$，最穩定且推薦的作法。

## 4. 常見錯誤與陷阱
1. **陣列長度**：應設為 `amount + 1`，否則存取 `dp[amount]` 會發生 `IndexError`。
2. **初始化**：
    - `dp[0] = 0` (Base case)。
    - 其他位置應初始化為 `float('inf')` 或 `amount + 1`。
3. **判斷無法湊成**：最後檢查 `dp[amount]` 是否仍為初始值，若是則回傳 `-1`。
4. **負數檢查**：在更新 `dp[i - coin]` 前，須確保 `i - coin >= 0`。
