# LeetCode 3. Longest Substring Without Repeating Characters 筆記

## 1. 題目解析
- **目標**：在字串中找出「不含重複字元」的最長**連續子字串**。
- **回傳值**：該子字串的長度（Integer）。
- **關鍵點**：子字串 (Substring) 必須連續，與子序列 (Subsequence) 不同。
- **測資特性**：字串可能包含英文字母、數位、符號與空格。

## 2. 核心演算法與資料結構
- **演算法**：**滑動視窗 (Sliding Window)**。
- **資料結構**：
    - **HashSet (集合)**：用於基礎法，記錄視窗內存在的字元。
    - **HashMap (雜湊表/字典)**：用於優化法，記錄 `字元: 最後出現的索引位置`。

## 3. 解題思路與方法比較

| 方法 | 實作方式 | 時間複雜度 | 空間複雜度 | 優缺點 |
| :--- | :--- | :--- | :--- | :--- |
| **暴力法** | 檢查所有子字串組合 | $O(n^3)$ | $O(min(n, m))$ | 簡單直觀但會 TLE |
| **基礎滑動視窗** | `right` 擴張，`left` 逐格收縮直到重複消失 | $O(n)$ | $O(min(n, m))$ | 邏輯好理解，每個字元進出一次 |
| **優化滑動視窗** | 利用 Map 紀錄位置，`left` 直接跳轉到重複字元的下一格 | $O(n)$ | $O(min(n, m))$ | 效能最優，減少 `left` 移動次數 |

> **優化法核心公式**：`left = max(left, last_seen_index + 1)`
> 這裡使用 `max` 是為了防止 `left` 往回跳到已經不在視窗內的舊索引。

## 4. 常見錯誤與陷阱
- **左邊界回跳 (The "abba" trap)**：這是最常見的 Bug。當遇到重複字元時，必須確認該字元的上一個索引是否還在當前 `left` 的右側。
- **計算長度的時機**：每次 `right` 右移後，不論是否發生跳轉，都應更新 `max_len = max(max_len, right - left + 1)`。
- **邊界情況**：空字串 `""` 應回傳 `0`；全重複字串 `"bbbbb"` 應回傳 `1`。
