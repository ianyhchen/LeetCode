# LeetCode 33. Search in Rotated Sorted Array 筆記

## 1. 題目解析
在旋轉後的有序陣列中尋找 `target`。由於陣列被旋轉，它由兩段遞增序列組成。目標是在 $O(\log n)$ 時間內完成搜尋。

## 2. 核心演算法與資料結構
* **演算法**：修正版二分搜尋 (Modified Binary Search)。
* **資料結構**：Array。

## 3. 解題思路與方法比較

### 方法一：One-Pass (推薦)
在一次二分搜尋中判斷哪一邊是有序的，並檢查 target 是否在該有序區間內。
* **優點**：效率最高，代碼簡潔。
* **缺點**：邏輯分支較多。

### 方法二：Two-Pass
1. 先用二分搜尋找到旋轉點（最小值）。
2. 判斷 target 在哪一個有序段。
3. 對該段執行標準二分搜尋。
* **優點**：模組化清晰，易於理解。
* **缺點**：需要處理兩次搜尋邏輯。

## 4. 關鍵細節：收斂邏輯與迴圈條件
| 類型 | 條件 | 說明 |
| :--- | :--- | :--- |
| **標準搜尋** | `while left <= right` | 確保檢查到最後一個元素，避免漏掉 target。 |
| **尋找邊界** | `while left < right` | 縮小範圍直到剩下一點，避免 `right = mid` 造成的死迴圈。 |

## 5. 常見錯誤與陷阱
* **死迴圈**：在尋找旋轉點時錯誤使用 `while left <= right`。
* **邊界遺漏**：判斷 target 區間時未考慮等號（如 `nums[left] <= target`）。
* **未旋轉處理**：代碼應能處理 `nums[0] < nums[n-1]`（即未旋轉）的情況。
