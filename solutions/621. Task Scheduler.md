# LeetCode 621. Task Scheduler 實作筆記

## 1. 題目解析
* 給定任務清單與冷卻時間 `n`，求完成所有任務的最短時間。
* 核心在於如何利用冷卻時間填入其他任務，以最小化 `idle` 狀態。

## 2. 核心演算法與資料結構
* **方法 A (Math)**: 利用最高頻率任務建立「桶子」框架，公式計算空位。
* **方法 B (Greedy)**: 使用 `Max-Heap` 追蹤剩餘次數最多的任務，搭配 `Queue` 處理冷卻時間。

## 3. 解題思路與方法比較
* **數學法**: 
  - 計算公式：`(max_freq - 1) * (n + 1) + n_max`。
  - 需與 `len(tasks)` 取最大值，以應對任務過多不需要 idle 的情況。
* **模擬法**: 
  - 每秒檢查是否有冷卻完畢的任務。
  - 優先執行剩餘次數最多的任務（Greedy Choice）。

## 5. 常見錯誤與陷阱
* **公式失效**: 當 $n$ 很小且任務種類很多時，公式可能小於總任務數。
* **冷卻計算**: 記得循環週期是 $n+1$，而非 $n$。
* **最後一組**: 公式中之所以是 `max_freq - 1`，是因為最後一組任務後方不接冷卻。
