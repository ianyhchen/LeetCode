## 1. 題目解析
判斷矩陣 `mat` 旋轉 0, 90, 180, 270 度後是否與 `target` 一致。

## 2. 核心演算法與資料結構
- **矩陣旋轉 (Matrix Rotation)**: 
  - 順時針 90 度 = 轉置 (Transpose) + 水平翻轉 (Reflect)。
  - 轉置公式: `matrix[i][j], matrix[j][i] = matrix[j][i], matrix[i][j]`
  - 水平翻轉公式: `row.reverse()` 或 `matrix[i][j], matrix[i][n-1-j] = matrix[i][n-1-j], matrix[i][j]`

## 3. 解題思路與方法比較
- **方法一：模擬旋轉**
  - 撰寫 `rotate` 函式，重複呼叫並與 `target` 比對。
  - 優點：邏輯模組化，容易理解。
  - 缺點：若非原地操作，空間複雜度 $O(n^2)$。
- **方法二：座標映射 (Index Mapping)**
  - 直接比對 `mat[r][c]` 與 `target` 在四種旋轉狀態下的對應位置。
  - 90度映射: `target[c][n-1-r]`
  - 180度映射: `target[n-1-r][n-1-c]`
  - 270度映射: `target[n-1-c][r]`
  - 優點：空間複雜度 $O(1)$。

## 5. 常見錯誤與陷阱
- **忽略 0 度**：必須先檢查原始矩陣是否就等於目標。
- **轉置範圍**：原地轉置時，內層迴圈必須從 `i+1` 開始，否則會交換兩次導致回到原狀。
- **Python List Copy**: 使用 `mat[:]` 僅是淺拷貝，處理二維陣列旋轉時若需備份，建議使用 `[row[:] for row in mat]`。
