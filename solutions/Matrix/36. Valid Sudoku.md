# LeetCode 36. Valid Sudoku (有效的數獨) - 核心筆記

## 1. 題目解析
- **任務**：判斷一個 $9 \times 9$ 的數獨板是否合法。
- **合法定義**（僅針對已填入數字）：
    1. **每行** (Row) 數字 1-9 不重複。
    2. **每列** (Column) 數字 1-9 不重複。
    3. **每個 3x3 九宮格** (Sub-box) 數字 1-9 不重複。
- **關鍵理解**：
    - 忽略空白格 `.`。
    - 「有效」不代表「有解」，不需考慮未來是否能填完，只需驗證現狀。

## 2. 核心演算法與資料結構
- **資料結構**：**哈希表 (Hash Set)** 或 **位元遮罩 (Bitmask)**。
- **演算法**：**單次遍歷 (One-pass Traversal)**。
    - 遍歷一次 $9 \times 9$ 矩陣，時間複雜度為 $O(1)$（因為棋盤大小固定為 81 格）。
    - 空間複雜度為 $O(1)$（固定數量個集合或整數）。

## 3. 解題思路與方法比較

### A. 列表生成式 (List of Sets) - 最推薦
- **做法**：建立三個列表，分別存放 9 個 `set()`，對應行、列與九宮格。
- **優點**：結構化強，存取直覺（如 `rows[r]`），是面試中最穩健的寫法。

### B. 單一集合標籤法 (Single Set with Tagging)
- **做法**：只用一個 `seen = set()`，存入帶有語義的元組，如 `(r, "row", val)`。
- **優點**：代碼極簡，非常有 Python 風格。

### C. 九宮格座標轉換 (關鍵邏輯)
- **公式**：`box_index = (r // 3) * 3 + (c // 3)`
- **推導**：
    - `r // 3`：判斷在哪一組「大橫排」（0, 1, 2）。
    - `* 3`：每一組大橫排包含 3 個九宮格，作為基礎偏移。
    - `c // 3`：判斷在該大橫排中的第幾個九宮格。



## 4. 常見錯誤與陷阱 (Common Pitfalls)

### 陷阱一：物件參考錯誤 (Object Reference Trap)
- **錯誤初始化**：`rows = [set()] * 9`
- **結果**：這會讓 9 個位置都指向**同一個集合物件**。修改 `rows[0]` 會導致所有 row 都被修改。
- **正確做法**：使用列表生成式 `[set() for _ in range(9)]`。

### 陷阱二：單一集合法的元組碰撞 (Tuple Collision)
- **問題**：若只存 `(index, val)`，會無法區分 `(0, "5")` 是代表「第 0 行有 5」還是「第 0 列有 5」。
- **修正**：必須在元組中加入標籤字串，如 `(index, "row", val)`。

### 陷阱三：座標運算順序
- **錯誤**：`r // (3 * 3) + c // 3` 等錯誤變體。
- **提醒**：務必記住公式是將維度先降階（除以 3），再進行維度組合。

### 陷阱四：忽略空白格判斷
- 在檢查前必須先判斷 `if board[r][c] == ".": continue`，否則會將 `.` 存入集合導致邏輯錯誤。
