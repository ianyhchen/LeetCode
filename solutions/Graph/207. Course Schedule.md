## 1. 題目解析
本題 (LeetCode 207) 核心在於判斷一個**有向圖**中是否存在**環 (Cycle)**。若存在環，則無法完成所有課程（即無法完成拓撲排序）。

## 2. 核心演算法與資料結構
* **資料結構：**
    * `Adjacency List`: 儲存 `src -> list of dest` 的鄰接表。
    * `In-degree Array`: 紀錄每個節點的入度（指向該節點的邊數）。
    * `Queue`: 存放目前入度為 0、可以立即處理的節點。
* **演算法：Kahn's Algorithm (BFS)**
    * 透過不斷移除入度為 0 的節點來達成拓撲排序。

## 3. 解題思路與方法比較
### BFS (Kahn's Algorithm)
1.  統計所有節點的 **In-degree**。
2.  將所有 **In-degree == 0** 的節點加入 Queue。
3.  當 Queue 不為空：
    * 取出節點 `u`，計數器 `count + 1`。
    * 遍歷 `u` 的所有鄰居 `v`，將 `in_degree[v] - 1`。
    * 若 `in_degree[v]` 變為 0，將 `v` 加入 Queue。
4.  最終若 `count == numCourses` 則無環。

### DFS (State Marking)
1.  使用三種狀態：`0 (Unvisited)`, `1 (Visiting)`, `2 (Visited)`。
2.  若在遞迴中遇到狀態 `1` 的節點，表示偵測到環。

| 方法 | 優點 | 缺點 |
| :--- | :--- | :--- |
| **BFS** | 邏輯直觀，易於擴展輸出排序結果 | 需額外維護入度表 |
| **DFS** | 程式碼較簡潔 | 遞迴深度過大可能導致 Stack Overflow |

## 5. 常見錯誤與陷阱
* **入度計算錯誤**：建立圖時方向畫反（應該是先修課指向後續課程）。
* **環的判定基準**：在 BFS 中，若最後處理的節點數小於總數，即代表有環。
* **孤立節點**：入度本來就是 0 的孤立課程也必須一開始就放入 Queue。
