# LeetCode 310. Minimum Height Trees 筆記

### 1. 題目解析
- **核心目標**：在給定的無向樹中，找出所有能使樹高度最小的根節點。
- **關鍵性質**：
  - 最小高度樹的根節點（Tree Center）只可能會有 1 個或 2 個。
  - 這些根節點必定坐落於樹的最長路徑（直徑）的中點。

### 2. 核心演算法與資料結構
- **演算法**：剝洋蔥式拓撲排序 (Onion Peeling Topological Sort)。
- **資料結構**：
  - `Adjacency List (Set)`：存儲圖結構並方便 $O(1)$ 刪除邊。
  - `Degree Array`：記錄每個點的度數，識別葉子節點（degree == 1）。
  - `Queue`：存儲每一層待處理的葉子節點。

### 3. 解題思路與方法比較
- **暴力法 $O(n^2)$**：對每個點跑 BFS/DFS，會逾時。
- **拓撲排序法 $O(n)$**：
  1. 處理特例：若 $n=1$ 直接回傳 `[0]`。
  2. 建立鄰接表並找出所有度數為 1 的葉子。
  3. 不斷移除當前所有葉子，並更新其鄰居度數。
  4. 當剩餘節點數 $\le 2$ 時，剩下的節點即為答案。

### 4. 常見錯誤與陷阱
- **$n \le 2$ 的邊界情況**：當節點數極小時，循環邏輯可能未觸發。
- **同步更新問題**：必須以「層」為單位處理葉子，不能一邊刪一邊把新產生的葉子加入當前層。
- **空間效率**：避免使用鄰接矩陣，應使用鄰接表以節省空間。
