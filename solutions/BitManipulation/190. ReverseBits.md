# LeetCode 190. Reverse Bits

## 1. 題目解析
- **目標**：給定一個 32 位元的無符號整數，將其二進位位元進行反轉（Reverse）。
- **輸入輸出**：輸入為整數，輸出亦為反轉後的整數。
- **重點**：即使輸入的數字較小，也要視為完整的 32 位元處理（例如：`1` 反轉後不是 `1`，而是 $2^{31}$）。

## 2. 核心演算法與資料結構
- **核心概念**：位元運算 (Bit Manipulation)。
- **關鍵操作**：
    - `n & 1`：取出 `n` 的最低位 (Least Significant Bit)。
    - `res << 1`：左移位元，為新位元騰出空間。
    - `res | bit`：合併位元。
    - `n >> 1`：右移位元，丟棄已處理的位元。

## 3. 解題思路與方法比較
### 方法一：逐位反轉 (Standard Approach)
- **邏輯**：使用一個 32 次的循環，每次從輸入 `n` 的尾部取出一位，放進結果 `res` 的尾部，並不斷移動 `res`。
- **複雜度**：時間 $O(1)$ (固定 32 次)，空間 $O(1)$。
- **面試推薦**：此方法最直觀，且在 Python 中最容易實作。

### 方法二：分治法 (Divide and Conquer)
- **邏輯**：逐層交換位元區塊（16位交換、8位交換...）。
- **複雜度**：時間 $O(\log 32)$。
- **特點**：適合硬體層級優化，但代碼較難記憶。

## 5. 常見錯誤與陷阱
- **未跑滿 32 位元**：若使用 `while n > 0`，會導致高位元的 `0` 沒有被正確反轉到低位元。
- **溢位問題**：在 C++/Java 中要注意 Unsigned Integer 的處理，Python 則需確保結果不超過 $2^{32}-1$（本題通常沒問題）。
- **順序錯誤**：注意 `res << 1` 與 `res | bit` 的順序，若先 OR 再左移，最後一次會多移一位。
