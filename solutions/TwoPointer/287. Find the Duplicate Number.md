# LeetCode 287. Find the Duplicate Number - Implementation Note

## 1. 題目解析
本題的核心在於將陣列映射關係 `i -> nums[i]` 視為鏈結串列。由於存在重複數字，必然存在多個節點指向同一個位置，形成一個有環的圖結構。

## 2. 核心演算法與資料結構
- **資料結構**: 隱性鏈結串列 (Implicit Linked List)。
- **核心演算法**: 佛洛伊德循環檢測 (Floyd's Cycle-Finding Algorithm)。
  - **階段一 (Intersection)**: 使用快慢指標找到環中的任一相遇點。
  - **階段二 (Entrance)**: 將一指標重置於起點，兩者同步進，再次相遇即為環入口（重複數）。

## 3. 解題思路與方法比較
- **快慢指標 (Optimal)**: $O(n)$ 時間, $O(1)$ 空間。不修改原陣列，是最符合面試限制的解法。
- **位元運算**: $O(n \log n)$ 時間, $O(1)$ 空間。邏輯穩定但效能略遜於快慢指標。
- **二分搜尋**: $O(n \log n)$ 時間, $O(1)$ 空間。基於數值範圍進行統計，適合在無法將問題抽象為環時使用。

## 4. 常見錯誤與陷阱
- **Index Out of Bounds**: 雖然題目保證範圍，但實作時需確保 `nums[nums[hare]]` 的合法性。
- **起點選擇**: 必須從 `index 0` 開始，且第一步必須手動觸發（或使用 `do-while`），否則 `while slow != fast` 會因為初始值相同而直接跳過。
